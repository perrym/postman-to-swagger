What does it do?

  • Converts Postman v2.1 to OpenAPI 3.0 (JSON/YAML).
  • Normalizes paths: /:id and {{var}} → {id}.
  • Merges query parameters (from URL + Postman fields) without duplicates.
  • Detects security (Bearer/Basic/API Key) and sets securitySchemes.
  • Includes response examples and infers minimal schemas.

Quick start

python postman-to-swagger.py   --postman ./Collection.postman_collection.json   --output ./out/openapi   --json-only

Result: openapi.json. Install PyYAML and omit --json-only for YAML.

With environment & strict mode

python postman-to-swagger.py   --postman ./Collection.postman_collection.json   --env ./MyEnv.json   --var baseUrl=https://api.example.com   --output ./out/openapi   --json-only --strict

CLI options

OptionDescription
--postman <path>Required. Postman collection (v2.1) JSON.
--env <path>Optional. Postman environment JSON; enabled keys feed server variables.
--var key=valueOptional, repeatable. CLI overrides for variables.
--base-url <url>Force the server URL; overrides auto-detection.
--folder <name>Convert only items whose folder name contains this substring.
--json-only / --yaml-only / --bothSelect output format(s).
--strictFail if {{var}} remains in URLs/paths.
--log-levelDEBUG, INFO, WARNING, ERROR.

Tips & pitfalls

  • Always export as Collection v2.1 in Postman.
  • Multiple hosts? Run per subfolder or use --base-url.
  • Placeholders in the server URL? Use --env or --var, or set --base-url.
  • Complex schemas? The converter infers minimal schemas; verify critical payloads manually.
